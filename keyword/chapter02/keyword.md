- RESTful API란?
    
    > REST의 원리를 따르는 API를 말한다.
    > 
    
    “HTTP 프로토콜을 사용하여 자원을 정의하고, URI와 HTTP 메소드로 통신을 주고받는 아키텍처”
    
    **[REST의 구성 요소]**
    
    - 자원(Resource)
    - 행위(Verb): 자원에 대한 행위
    - 표현(Representations): 자원에 대한 행위의 내용
    
    **[REST의 특징]**
    
    - 서버-클라이언트 구조
    - 무상태
    - 캐시 가능
    - 계층화
    - 인터페이스 일관성
    
    **[REST API 설계 규칙]**
    
    - URI는 명사와 소문자 사용
    - 마지막에 슬래시(/)를 사용하지 않는다
    - URI에서 단어의 구분이 필요한 경우 -(하이픈)을 사용
    - 자원은 기본적으로 복수형으로 표현
    - 행위를 포함하지 않는다
    

- 멱등성이란?
    
    > 동일한 연산을 여러 번 적용하더라도, 결과가 처음 1번 적용했을 때와 동일하게 유지되는 성질
    > 
    
    **[HTTP 메소드 멱등성]**
    
    - GET: 리소스를 조회
        - 멱등성 O
        - 여러번 조회해도 리소스는 변하지 않음
    - POST: 리소스를 생성 또는 처리
        - 멱등성 X
        - 새로운 리소스를 생성하거나 리소스 상태가 달라지면서 호출 결과가 달라질 수 있음
    - PUT: 리소스를 대체
        - 멱등성 O
        - 여러번 호출해도 항상 대상 리소스를 대체하여 동일한 상태
    - PATCH: 리소스를 수정
        - 멱등성 X
        - PATCH는 멱등하다고 생각할 수 있으나, 범용적 사용되기 때문
        - 기존 리소스에 “값을 추가(append)”하는 경우에도 사용될 수 있어, 이 경우 결과가 달라질 수 있음
    - DELETE: 리소스를 삭제
        - 멱등성 O
        - 여러번 호출해도 리소스가 없는 상태
    

- HTTP 메서드 종류
    
    > GET, POST, PUT, PATCH, DELETE
    > 
    
    - GET: 리소스를 조회
    - POST: 리소스를 생성 또는 처리
    - PUT: 리소스를 대체
    - PATCH: 리소스를 수정
    - DELETE: 리소스를 삭제
